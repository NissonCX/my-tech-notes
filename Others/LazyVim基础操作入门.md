# LazyVim基础操作入门

日期：2026-04-19
标签：LazyVim, Neovim, 编辑器, 开发工具

## 问题
我在使用 LazyVim 时遇到很多问题：写好代码后不知道如何保存、如何给代码文件命名、如何格式化代码、如何撤回操作、如何快速删除不想要的代码、如何在目录树里切换文件，以及误触分屏后如何恢复。

## 回答
LazyVim 对新手最重要的是先掌握“模式切换”和“几个高频命令”。

### 1. 先分清两种模式

- 插入模式：真正输入文字，按 `i` 进入
- 普通模式：执行命令、跳转、删除、保存，按 `Esc` 返回

如果不确定当前该怎么操作，先按一次 `Esc`，再执行命令，通常最稳。

### 2. 如何保存代码并给文件命名

当前编辑的是未命名缓冲区时，先按 `Esc`，再输入：

```vim
:w Singleton.java
```

回车后会把当前内容保存为 `Singleton.java`。

如果要直接保存到某个目录：

```vim
:w coding-test/Singleton.java
```

常用保存命令：

- `:w` 保存
- `:wq` 保存并退出
- `:q!` 不保存强制退出

### 3. 文件和文件夹怎么创建、重命名

在左侧目录树中：

- 按 `a` 新建文件或文件夹
- 输入 `java/` 这种以 `/` 结尾的名字表示新建文件夹
- 输入 `Singleton.java` 表示新建文件
- 选中文件后按 `r` 表示重命名

### 4. 如何撤回和重做

在普通模式下：

- `u` 撤回
- `Ctrl + r` 重做

### 5. 如何更快删除代码

不要再一个字符一个字符删，普通模式里直接用这些命令：

- `x` 删除一个字符
- `dd` 删除当前整行
- `3dd` 删除当前开始的 3 行
- `dw` 删除一个单词
- `D` 删除从光标位置到行尾

### 6. 如何切换到目录树并打开其他文件

常用操作：

- `空格 e` 打开或关闭左侧目录树
- `Ctrl + h` 从代码区跳到左侧目录树
- `j` / `k` 在目录树中上下移动
- `Enter` 打开选中的文件
- `Ctrl + l` 从左侧目录树回到右侧代码区

如果想在已打开文件之间快速切换，也可以试：

- `空格 ,`

这通常会打开缓冲区/最近文件列表。

### 7. 误触分屏后怎么恢复

如果不小心出现上下或左右分屏：

- `:q` 关闭当前窗口
- `:only` 关闭其他分屏，只保留当前窗口

在多个窗口之间切换：

- `Ctrl + h` 到左边窗口
- `Ctrl + l` 到右边窗口
- `Ctrl + j` 到下面窗口
- `Ctrl + k` 到上面窗口

### 8. 如何格式化 Java 代码

在普通模式下按：

```text
空格 c f
```

这是 LazyVim 里常见的格式化快捷键。

如果提示 `No formatter available`，通常说明：

- 当前不是实际的 `*.java` 文件
- Java 格式化器没有安装
- LazyVim 没有为 `java` 文件类型接上格式化工具

可用下面的命令排查：

```vim
:set filetype?
:ConformInfo
```

如果 `:ConformInfo` 显示当前缓冲区 formatter 是 `<none>`，就说明当前文件还没有可用格式化器。

## 相关问题
- LazyVim 里如何保存并给文件命名？→ 先按 `Esc`，再用 `:w 文件名`，例如 `:w Singleton.java`
- LazyVim 里如何创建文件夹和新文件？→ 在目录树里按 `a`，输入 `文件夹名/` 创建文件夹，输入 `文件名` 创建文件
- LazyVim 里如何重命名文件？→ 在目录树里选中文件后按 `r`
- LazyVim 里如何撤回和重做？→ `u` 撤回，`Ctrl + r` 重做
- LazyVim 里如何快速删除代码？→ `x` 删除字符，`dd` 删除整行，`dw` 删除单词
- LazyVim 里如何切到目录树打开其他文件？→ `Ctrl + h` 切到左侧目录树，`j/k` 移动，`Enter` 打开文件
- LazyVim 里误触分屏后怎么恢复？→ `:q` 关当前窗口，`:only` 只保留当前窗口
- LazyVim 里 Java 文件为什么不能格式化？→ 多半是当前不是实际 Java 文件，或者还没给 `java` 配置格式化器

## 技术拓展
LazyVim 本质上还是 Neovim 的一套预配置，因此很多操作依然遵循 Vim 的核心思路：普通模式负责“操作文本”，插入模式负责“输入文本”。一旦熟悉 `Esc`、`:w`、`u`、`dd`、`Ctrl + h/j/k/l` 这些高频动作，日常编辑效率会比图形化编辑器里的鼠标操作更高。
